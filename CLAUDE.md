# ZELEX — Claude Project Instructions

## Project Overview

**ZELEX Character Atlas** — a static, data-driven catalog site for the ZELEX doll collection. Bodies are classified into 6 Body Families (Classic · Icon · Muse · Siren · Empress · Sculpt) using WHR/BWR measurements. Characters are generated from a live product feed + spec-card data.

This is a **private brand project** (proprietary, not open source).

---

## Stack

- **Frontend**: Vanilla HTML/CSS/JS — no framework, no build step
- **Shared assets**: `assets/site.css` + `assets/site.js` (global `ZX` object)
- **Data**: `db/*.json` — generated by Python pipeline, hand-curated via `db/character_overlay.json`
- **Server**: Caddy (local dev via `Caddyfile`) or `serve.py` (Python fallback)
- **Pipeline**: Python scripts in `scripts/` — run in order (see below)

---

## Running Locally

### Caddy (preferred)
```bash
caddy run          # serves howiez.local — requires hosts entry
```

Add to `C:\Windows\System32\drivers\etc\hosts`:
```
127.0.0.1  howiez.local
```

### Python fallback
```bash
python serve.py    # http://localhost:8000
```

---

## Data Pipeline (run order)

```
build_db.py          # scan assets + live feed → catalog.db + catalog.json
build_profiles.py    # WHR/BWR analysis → classify into 6 Body Families
build_characters.py  # Series → Body → 4 characters → photoshoot|placeholder
merge_stories.py     # fold story/profile inputs into characters.json
make_thumbs.py       # generate thumbnails (re-run after build_characters)
build_package.py     # stage deliverable (code + images) + zip
```

## Key Config (before launch)

- `FORM_ENDPOINT` — Formspree/Getform URL (falls back to mailto if empty)

# ZELEX Community Hub — Brand Team Training Guide

**Version:** 1.0
**Launch Date:** 2026-06-21
**Target Audience:** Moderation team, brand team, community managers

---

## Overview

The Community Hub is a three-pillar system designed to amplify collector voice while maintaining brand safety and quality standards:

1. **Gallery** — Verified collector photo submissions with moderation queue
2. **Events Calendar** — Community events with calendar subscription (iCal)
3. **Reviews** — Verified purchase reviews featured in carousel

This system targets **100+ submissions** in the first quarter post-launch.

---

## Pillar 1: Community Gallery

### Purpose
Showcase collector creativity, styling tips, and setup inspiration while maintaining brand safety and quality.

### Key Features

| Feature | Details |
|---------|---------|
| **Submission Form** | Link: `contact.html?src=gallery&cta=submit_photo` |
| **Moderation Queue** | Reviewed within 24 hours; approval/rejection tracked |
| **Featured Showcase** | Top submissions appear in carousel on homepage and social |
| **Body Family Filtering** | Gallery filterable by Classic/Icon/Muse/Siren/Empress/Sculpt |
| **Data File** | `db/community_gallery.json` — submission metadata + ratings |

### Moderation Workflow

1. **Submission Received** → form endpoint captures photo + metadata
2. **24-Hour Review** → brand team evaluates for:
   - Brand alignment (ZELEX products in good light)
   - Content safety (no explicit/hateful/political content)
   - Photography quality (clear lighting, focus)
   - Collector verification (verified purchase flag checked)
3. **Approval → Gallery** → thumbnail generated, added to `community_gallery.json`
4. **Rejection → Private Feedback** → contact form reply with reason + guidance for resubmission

### Common Moderation Scenarios

#### ✅ APPROVE
- Professional lighting + clear product focus
- Verified collector with positive history
- Descriptive title/tags that help categorization
- Regional diversity encouraged
- Creativity + personalization celebrated

#### ❌ REJECT
- Blurry/dark photos difficult to see product details
- Unverified collector or brand safety concern
- Political/adult/hateful content or harassment
- Watermarked content from other brands
- Spam or self-promotion beyond description

#### 🟡 REQUEST CHANGES
- Excellent photo but needs retagging for accuracy
- Minor light/contrast adjustment recommended
- Ask for regional context to improve searchability
- Request photo featuring multiple bodies for comparison

### Featured Spotlight Process

**Selection Criteria:**
- 50+ likes minimum
- Verified collector
- 3+ weeks in gallery
- Unique perspective or professional quality
- Regional diversity

**Promotion Path:**
1. Approve for featured carousel on `gallery.html`
2. Resize to 1200x1200px; upload to CDN
3. Share to Instagram/TikTok (@zelexdoll, tag creator)
4. Monthly digest email to subscriber list

## Pillar 2: Events Calendar

### Purpose
Build community through recurring events (onboarding, workshops, meetups) and enable calendar subscriptions.

### Key Features

| Feature | Details |
|---------|---------|
| **Calendar File** | `db/community_events_calendar.json` — 6+ recurring events |
| **iCal Export** | Generated by `scripts/generate_ical.py`; published at `/community-calendar.ics` |
| **Subscription** | Users add to Google Calendar, Outlook, Apple Calendar |
| **RSVP Tracking** | Current/max attendance tracked per event |
| **Recurring Rules** | MONTHLY / BI-MONTHLY / QUARTERLY patterns |

### Event Types

#### 1. Onboarding Office Hours (Monthly)
- **When:** First Sunday, 14:00 UTC / 9:00 AM EST / 6:00 AM PST
- **Where:** Discord #onboarding
- **Lead:** Community manager
- **Topics:** New collector orientation, channel etiquette, moderation, verification
- **RSVP:** Via contact form → private Discord invite

#### 2. Magazine Preview (Monthly)
- **When:** Second Sunday, 16:00 UTC / 11:00 AM EST / 8:00 AM PST
- **Where:** Zoom (link sent to registered attendees)
- **Lead:** Editorial team
- **Topics:** Upcoming issue preview, story submission, cover voting
- **RSVP:** Magazine waitlist signup

#### 3. Safety + Verification AMA (Bi-monthly)
- **When:** Third Sunday, 15:00 UTC / 10:00 AM EST / 7:00 AM PST
- **Where:** Discord #ama-general
- **Lead:** Moderation team
- **Topics:** Handle verification, impersonation detection, reporting flow
- **RSVP:** AMA question submission

#### 4. Photography Workshop (Monthly)
- **When:** Fourth Sunday, 13:00 UTC / 8:00 AM EST / 5:00 AM PST
- **Where:** YouTube Live + Discord watch party
- **Lead:** Photography expert (freelance or contractor)
- **Topics:** Lighting, camera settings, composition, post-processing
- **RSVP:** Workshop registration form

#### 5. Customization Show & Tell (Monthly)
- **When:** Saturday after 1st, 17:00 UTC / 12:00 PM EST / 9:00 AM PST
- **Where:** Discord #showcase-gallery
- **Lead:** Community moderator
- **Topics:** Custom mods, materials, techniques, gallery highlights
- **RSVP:** Gallery submission form

#### 6. Regional Meetups (Quarterly)
- **When:** 2nd Friday (dates vary by region)
- **Where:** In-person meetups (multiple cities)
- **Lead:** Regional coordinators
- **Topics:** Meet-and-greet, setup showcase, exclusive merch
- **RSVP:** Separate registration per region

### iCal Subscription Setup

**For End-Users:**
1. Copy link: `https://www.zelexdoll.com/community-calendar.ics`
2. Google Calendar: Add via "Add by URL" (+ button → Subscribe to calendar)
3. Outlook: Add via "Add Calendar" → From Internet
4. Apple Calendar: Add via File → Import

**For Brand Team (Regenerating):**
```bash
python scripts/generate_ical.py
# Output: ./community-calendar.ics (260+ bytes)
# Deploy to: /community-calendar.ics (CDN or root)
```

### Event Lifecycle

1. **Define** → Add to `community_events_calendar.json` with all fields
2. **Generate** → Run `scripts/generate_ical.py` to export .ics
3. **Promote** → Post on social, send to email list 2 weeks ahead
4. **Execute** → Host live; record if applicable
5. **Archive** → Mark completed; keep for historical reference
6. **Feedback** → Send survey to attendees; gather improvement notes

## Pillar 3: Reviews & Testimonials

### Purpose
Build trust through verified customer feedback; feature in homepage carousel and product pages.

### Key Features

| Feature | Details |
|---------|---------|
| **Verified Purchases** | Flag set to `true` only if purchase verified via order system |
| **Rating System** | 1-5 stars; aggregated for AggregateRating schema |
| **Featured Carousel** | Auto-rotates on `gallery.html` + homepage |
| **Helpfulness Tracking** | Count readers who found review helpful (future feature) |
| **Data File** | `db/community_gallery.json` → `reviews[]` array |

### Review Submission Flow

1. **User fills review form** → accessible from `gallery.html` + product pages
2. **Metadata collected:**
   - Rating (1-5 stars, required)
   - Title (max 100 chars)
   - Body (500-2000 chars)
   - Body family (dropdown)
   - Verified purchase (auto-flagged or unchecked if unverified)
3. **Moderation** → checked for spam/harmful content (same as gallery)
4. **Published** → added to `community_gallery.json`; featured if 4+ stars

### Review Carousel UI

**Homepage Banner:**
```html
<div id="reviews-carousel"></div>
<script>
  GalleryWidget.renderReviewCarousel(reviewsData, 'reviews-carousel');
</script>
```

**Auto-rotation:** Cycles every 6 seconds
**Manual navigation:** Click dots to jump to specific review

### Encouraging Reviews

- Email post-shipment: "Share your unboxing video or photos"
- Discord: Monthly "Review Spotlight" thread
- Gallery featured submissions: Offer review spotlight opportunity
- Incentive: Featured reviews get 10% discount on next order (optional)

## Moderation Team Roles

### Gallery Moderator
- Responsibilities:
  - Review submissions within 24 hours
  - Approve/reject with feedback
  - Tag submissions with body family + region
  - Nominate monthly featured selections
- Frequency: 1-2 hours daily

### Events Coordinator
- Responsibilities:
  - Schedule + coordinate monthly events
  - Send calendar reminders (2 weeks, 3 days, 1 day before)
  - Record + archive event streams
  - Manage RSVP lists + regional meetup logistics
- Frequency: 4-6 hours weekly

### Community Manager
- Responsibilities:
  - Overall hub strategy + content planning
  - Social media promotion of gallery/events/reviews
  - Respond to moderation appeals
  - Track metrics + report to leadership
- Frequency: 6-8 hours weekly

## Escalation Path

**Inappropriate Submissions:**
- Scam/impersonation → Forward to legal + moderation
- Harassment/threats → Screenshot, block user, report
- Adult content → Immediate rejection + user warning
- Repeated violations → User ban + notify collectors

**Appeals:**
- Collector can request reconsideration within 7 days
- Manager reviews with 2nd moderator
- Final decision within 3 business days

## FAQ

**Q: How do I submit a photo?**
A: Click "Submit Photo" button on gallery.html; form goes to contact endpoint; reviewed within 24 hours.

**Q: Can I edit a submission after it's approved?**
A: No, but you can delete and resubmit; contact moderation if you need to change approved submission details.

**Q: What if my submission is rejected?**
A: You'll receive an email with feedback and guidance; most rejections are fixable (lighting, framing, metadata).

**Q: How do I subscribe to the events calendar?**
A: Copy the .ics link from community-events.html; add to Google/Outlook/Apple Calendar.

**Q: Can I view the moderation queue?**
A: Only if you're on the moderation team; request access via contact.html with "join_moderation" CTA.

**Q: Are reviews anonymous?**
A: No; reviews are tied to your collector profile. Verified purchase flag helps build trust.

**Q: How often are events held?**
A: Monthly recurring core events (onboarding, workshop, show & tell); bi-monthly for AMA; quarterly for regional meetups.

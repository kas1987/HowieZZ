Accessibility Features

All components are built to WCAG AA standards with semantic HTML, keyboard navigation, and assistive technology support.

Focus Indicators
Gold outline with 3px offset on all interactive elements. High contrast against both light and dark backgrounds.
And links too
:focus-visible { outline: 2px solid var(--gold); outline-offset: 3px; }
Color Contrast
All text meets WCAG AA standards (4.5:1 for small text, 3:1 for large). Non-red/green-dependent status indicators.
Text on background
High contrast (#e8e8e8 on #121212)
Muted text
Still meets 4.5:1 (#ababab on #121212)
Semantic HTML
Proper heading hierarchy, form labels, ARIA labels, and list structures for screen readers.
<!-- ✓ Good --> <button class="btn" type="button">Submit</button> <label for="email">Email</label> <input id="email" type="email"> <!-- ✗ Avoid --> <div class="btn" onclick="...">Submit</div> <input placeholder="Email">
Keyboard Navigation
All interactive elements are keyboard accessible. Tab order follows visual flow.
• Tab: Move to next interactive element • Shift+Tab: Move to previous element • Enter: Activate button or submit form • Space: Toggle checkboxes or activate buttons • Arrow keys: Navigate select options • Escape: Close modals (if implemented)
Screen Reader Support
ARIA labels, roles, and live regions for assistive technology users.
<!-- Status indicators --> <span class="stat stat-live" aria-label="Live Status"> Live </span> <!-- Modals --> <div role="dialog" aria-labelledby="title"> <h3 id="title">Confirm</h3> </div> <!-- Skip link --> <a href="#main" class="skip-to-main"> Skip to main content </a>
Reduced Motion
Respects prefers-reduced-motion media query. Animations are disabled for users who prefer it.
@media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }
Form Validation
Error messages are associated with inputs via aria-describedby. Hints and errors are always visible, never hidden.

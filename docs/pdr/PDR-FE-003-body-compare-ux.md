# PDR-FE-003: Body Compare UX

## Objective
Design a first-class body comparison experience so buyers can compare body architecture before inquiry.

## Scope
Claude Design should create/refine:
- `compare.html` layout
- empty state
- 2-4 body comparison state
- body selector visual design
- family badges
- buyer-read row
- handling-confidence row
- estimated/verified state styling
- CTA to concierge/contact

## Acceptance criteria
- Compare table is readable on mobile.
- Buyer can understand differences without knowing ratio math.
- Missing/estimated data does not look broken.
- Contact handoff CTA is visually clear.

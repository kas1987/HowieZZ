# PDR-FE-007: Browse and Family UX

## Objective
Improve browse/family pathways so buyers can navigate by series, family, body architecture, and character without confusion.

## Scope
Claude Design should improve:
- family landing page hierarchy
- browse filter visual treatment
- card metadata density
- family badges
- live/concept/shoot-pending states
- empty states for undercovered families
- CTA path from browse/family to compare/contact

## Acceptance criteria
- Users can understand family vs series vs character.
- Undercovered families are not presented as broken.
- Buyer can move from browse/family to compare/contact.

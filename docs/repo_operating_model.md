# Repo Operating Model — ZELEX Character Atlas

## Source of Truth

`main` is the source of truth for production-ready code. The live GitHub Pages site deploys from `main` on every push.

## Branching

Use short-lived branches off `main`:

```text
feature/<short-description>
bugfix/<short-description>
hotfix/<short-description>
docs/<short-description>
chore/<short-description>
```

Prefer small, focused branches. One purpose per PR.

## Pull Requests

All changes go through a PR. Every PR should include:

- Clear summary of what changed and why
- Type (feature / fix / data / tooling)
- Checklist items (see `.github/PULL_REQUEST_TEMPLATE.md`)
- Screenshots or logs where relevant

## Review Gate

Before merge:

- CI passes (`validate` job)
- No images, video, or confidential docs in the diff
- Shared kit (`assets/site.css` / `assets/site.js`) changes reviewed carefully — all pages depend on them
- Data changes regenerated by pipeline, not hand-edited

## CI Gate

CI runs on pull requests and pushes to `main`. It validates:

- Python build scripts compile without error
- Site pages, inline JS, and catalog JSON are valid
- Community data schema and provenance
- v2 HTML package is up to date
- Analytics sanity suite passes
- No binary media or confidential docs committed

## Deploy

GitHub Pages deploys automatically when the `validate` job passes on a push to `main`.

## Release

Tag stable milestones using semantic versions:

```text
v1.0.0   initial atlas release
v1.1.0   new series or major feature
v1.1.1   bugfix or content patch
```

Release notes should cover: Added · Changed · Fixed · Security.

## Stack Quick Reference

| Layer | Tech |
|-------|------|
| Frontend | Vanilla HTML / CSS / JS (`ZX` global) |
| Data | `db/*.json` generated by Python pipeline |
| Server (local) | Caddy → `https://howiez.local` |
| Server (prod) | GitHub Pages |
| CI | GitHub Actions |

## LLM Agent Rules

Agents may:
- Add or improve templates, docs, CI config
- Refactor within a scoped PR
- Generate or update catalog data via the pipeline scripts

Agents must not:
- Commit secrets, credentials, or `.env` files
- Delete files without explicit instruction
- Rewrite git history
- Add deployment steps without explicit instruction
- Hand-edit generated `db/*.json` (run the pipeline instead)
